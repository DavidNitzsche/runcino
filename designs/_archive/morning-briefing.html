5
Watch out

Known issues

All called out so they don't surprise you. None are blockers; all M1.

Math edge case
Last-phase pacing can produce unrealistic targets.

When the final phase is short (e.g. Sombrero's 1.16-mi "Finish strong") and the underlying GPX has even a slight downhill there, the Minetti GAP + total-time-balancing math can pin that phase at e.g. 6:05/mi. Goal-time invariant still holds. Fix is in lib/pacing.ts — likely a per-mile pace floor scoped to short trailing phases.

Visual debt
/training, /retrospective, /research, /settings/integrations look broken.

They were styled for the old light theme and reference classes that no longer exist (.faff-card, .btn-accent, --color-paper). Functionality intact, visuals degraded. Mechanical migration tracked in NEXT_PHASE.md.

Architecture
No server-side persistence.

Saved races live entirely client-side. SSR shows the empty state for a beat before hydration. Workable for a personal tool. Path to iCloud-backed sync proposed in NEXT_PHASE.md §1.

Tooling
Claude preview screenshot tool returns blank dark images mid-page.

Tool quirk, not a page bug. DOM queries confirm everything renders. Doesn't affect actual users — just slowed my visual verification last night.
