Shared notes
Rules of engagement
  1. Never push to main without David's explicit approval
  2. Never modify docs/SCHEMA.md without version-bumping (major bump breaks iOS)
  3. Never add a landmark without a primary-source citation — see web/data/courses/big-sur-marathon.json
  4. Commit style: imperative, specific, under 72 chars first line, body explains why
  5. Tests stay green. Fix the test or fix the change. Never comment-out tests.
  6. Claude API calls: use cache_control: { type: 'ephemeral' } on every system message
  7. Ask before guessing. David's time is the bottleneck.
  8. Mockups = real code. If you're writing HTML, stop — make it a React page in web/app/.
