faff · web today · states + rules · 2026-05-27

eleven states.
one pattern.

Pattern's locked: poster + sibling + week strip + mini-tiles. Now the work is showing what every state looks like under that pattern, and writing the rules that pick between them. Below: every state Faff supports, what triggers it, what fills the poster, what fills the sibling.
Each state has a resolver clause — when does this state fire — and a render spec — gradient, verb, sibling content, tiles. Build sprint 01 implements the resolver as a pure function; sprint 02 fills in the verb table + sibling variants; sprint 03 polishes the transitions.
the pattern · locked

three building
blocks.

poster · sibling · week strip — every state composes these
01
poster

The gradient card.

Eyebrow tag · verb (Bebas hero) · phase-tag (small caps) · stat trio at the bottom. Sometimes the stat trio is a choice button pair (MISSED) or a logistics block (RACE WEEK). Gradient = state.

02
sibling

The data card.

Dark card next to the poster. Header (At a glance / Run details / Why X / Setup) + optional prose + 2×N mini-tile dashboard. Tiles have a status dot (green/amber/red), big number, one-line meta. Same component, different content per state.

03
week strip

The arc.

Section header (THIS WEEK + mileage summary) + 7 day cards with colored accent bars. Bar color matches state. Today gets the white outline. Done days get the check badge. Always visible, never collapses.

implementation order

three sprints.
eleven states.
one resolver.

Sprint 01 · spine (1 week). Implement the resolver as a pure function returning one of the 11 states. Build the 3 components — Poster, Sibling, WeekStrip — accepting state + data. Ship 4 states: EASY · QUALITY · LONG · REST. These cover ~70% of daily home views.

Sprint 02 · post-run + edges (1 week). Add DONE · NAILED, DONE · EASE OFF, MISSED, NIGGLE. Verb table reaches ~25 phrases. Sibling card learns "Run details" and "Why move on" variants. Auto-promotion + 6h TTL on DONE state.

Sprint 03 · special states (1 week). SICK detection (5-day biometric rule). RACE WEEK takeover with race-day ops sibling. NEW USER onboarding with 4-step setup. Streak pause/resume logic. Hero takeover gesture.

Total: 3 weeks for the complete state coverage. All on the same SwiftUI/Next.js component substrate. Resolver is testable as a pure function — every state gets a unit test with hand-crafted fixture data. Each new edge case discovered post-ship becomes a new clause + a new test, not a new screen.
