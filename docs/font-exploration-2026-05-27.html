faff · font exploration · 2026-05-27

when type
breaks down.

Bebas Neue is heroic at 90px+. It does what no other free font does — shouts. But the same condensed letterforms that make RUN EASY. work at 120px start to crumble below 14px, exactly where workout details, tile labels, and rep tables need to live. Oswald has the same problem. Inter holds up at small sizes but lacks the heroic feel up top.
Below: the problem on a real card · twelve candidate fonts run through a size ladder · pairing rules · a proposed FAFF stack that fixes the legibility gap without losing the brand voice up top.

Problem 01 · condensed at small

Bebas Neue + Oswald are both condensed. At <14px the narrow letterforms blur together. "3:02" at 18px is fine but "800m" at 9px reads like static.

Problem 02 · no tabular nums

Bebas digits aren't tabular — the 3:02 across six tiles doesn't align cleanly. Data tables need fixed-width digits.

Problem 03 · all-caps fatigue

Bebas is caps-only. Used everywhere it shouts everywhere — robs the actual hero verbs of their punch. Reserve impact for impact moments.

02 · type primer

three jobs.
three fonts.

display · body · mono — never one font doing all three
Any serious type system splits into three roles. Display for the moments that should sing — hero verbs, race countdowns, single-number takeovers. Body for everything you actually read — descriptions, labels, prose, tile metadata. Mono / tabular for data alignment — tile grids, splits tables, code-like data where columns must line up. Pick the best font for each job, don't try to make one font do all three.

06 · the proposed stack

three fonts.
three jobs.
one system.

display for impact · Inter for body and small · Geist Mono for data
Keep Bebas Neue for what it's great at — the heroic moments. Promote Inter to handle small labels (replacing Oswald entirely) by leaning on its 700-800 weights with letter-spacing. Add Geist Mono for tabular data — rep grids, splits tables, anywhere digits must align column-to-column. Three fonts. Clear roles. No condensed letterforms below 24px.
DISPLAY

BEBAS NEUE

Use for impact moments only. Hero verbs (RUN EASY. · NAILED IT.) · race-week countdowns · big single numbers · poster eyebrows when caps-tracked. Hard floor: 24px. Below that, never.

Workhorse for everything readable. Body prose · tile metadata · descriptions · chip labels · small caps-tracked tags. Use weight + letter-spacing to differentiate roles. Inter at 700wt + 1.2px letter-spacing replaces Oswald entirely — same caps-tracked feel, way more legible at small sizes.

For data alignment. Rep grids · splits tables · per-mile pace columns · sync timestamps · code-like data display. Every digit the same width — eyes scan rows and columns cleanly. Pairs beautifully with both Bebas (above) and Inter (everywhere else).

pairing rules

When to use which.

  1. Bebas above the fold. Hero verbs, poster headlines, countdown numbers, section titles. Anywhere the goal is impact.
  2. Inter for everything in between. Body prose, tile metadata, chip labels — even the small caps-tracked tags that used to be Oswald. Inter at 700wt with 1-1.4px letter-spacing reads cleaner than Oswald at every size we use.
  3. Geist Mono only where data aligns. Rep grids, splits, mile-by-mile pace tables, sync timestamps. Don't use mono for narrative prose — it'll feel like a terminal.
  4. Inter font-variant-numeric: tabular-nums for any number 14px or smaller that lives inline with text. Inter has tabular figures built-in — turn them on.
  5. Drop Oswald entirely. Every place it currently lives (small labels, sub-headers, eyebrows) → Inter at 700wt with letter-spacing. Less condensed, more legible, fewer fonts to load.

What changed · same data

Header label. Was Oswald 10px caps-tracked. Now Inter 11px 600wt caps-tracked + 1.2px letter-spacing. Cleaner counter spaces, every letter visible.

Header value. Was Bebas 18px ("6.0 mi · ~48 min"). Now Inter 16px 600wt with tabular-nums. Numbers align, no condensed crush.

Segment labels (WU/CD/1-6). Was Oswald 8px. Now Inter 9px 700wt + 0.4px letter-spacing. Higher contrast strokes.

Per-rep "R1/R2/..." labels. Was Oswald 8px. Now Inter 9px 700wt caps + 1px letter-spacing. Easier to scan at a glance.

Rep values (3:02). Was Bebas 18px (no tabular). Now Inter 18px 700wt with tabular-nums. Every digit same width, column scan works.

Footer values. Was Bebas 18px. Now Inter 17px 700wt with tabular-nums. Less condensed, more honest numbers.

implementation · what changes

two new tokens · one font dropped.

Add: Geist Mono as --f-mono. Add inter weights 700 and 800 to the existing Inter load (they're already on Google Fonts CDN, just need to specify them).

Drop: Oswald entirely. Every place it currently lives — small labels, sub-headers, eyebrows, chips — switches to Inter at 700wt with letter-spacing 1.2-1.4px. Same caps-tracked feel, no condensed-letterform crush.

Restrict: Bebas Neue to ≥24px sizes only. Below that, switch to Inter at 700wt. Mostly affects card titles and small headers that were trying to be Bebas at 14-18px.

Adopt: Geist Mono for any data table, rep grid, splits chart, or place where digits must align column-to-column. Use font-variant-numeric: tabular-nums on Inter for inline numbers smaller than 14px.

Net effect: brand voice up top stays loud · the detail layer down below stops crushing · data finally aligns. Same emotional range, more honest at every size.
