What changed from v1

  • Coach decides what the page shows. The LLM returns both the prose voice AND a structured topics[] array of things it raised. Each topic → one card. If the coach didn't raise it, no card. The page reshapes per day based on the coach's reading.
  • "Research shows" — no named researchers in body. The voice cites findings without dropping "Heiderscheit" / "Daniels" inline. Names live in a separate audit trail you tap to expand.
  • Week strip moved to top. Anchors "where am I in the week" before reading the coach. Demoted to a thinner glance row (day-letter + mileage + completion dot, no type tag).
  • Readiness ring → small chip top-right. 88 inside a 44px ring next to the date. Tap to expand the trend rows (sleep / freshness / RHR) on a /health page. No giant ring eating the hero.
  • Reply chips → Bebas labels, no emojis. SOLID / TIRED / WRECKED with colored hover state. Matches the watch's "Done" / "START" pill style — letterforms carry the meaning.
  • "Let me know how it felt" — past tense, references the run that already happened.
  • Run recap demoted. Now at the bottom as supporting evidence, not the second hero. The 6-tile RUNNING FORM grid is gone from TODAY — lives behind the "View splits + route" link on the Run Detail page.

Topic library so far

Finite set. Renderer knows how to draw each kind. New kinds added as we hit them.

  • cadence_experiment · current/target SPM + research reason + try-tomorrow CTA
  • profile_gap · missing field + why coach needs it + inline input
  • sleep_deficit · 7-night avg + per-night bars + deficit vs target
  • recovery_amber · HRV/RHR flagged + concern summary
  • next_workout · upcoming session preview
  • week_shape · banked vs planned + tone (on_target / ahead / behind)
  • weight_trend · current + 30-day delta + direction
  • race_horizon · race name + days away + tone (comfortable / building / tightening / race_week)

What this means architecturally

  • One LLM call per cache-invalidation event. Returns voice + topics. ~$0.01.
  • Renderer is dumb. Maps each topic kind to a React component. No business logic.
  • The coach is the eye. Tomorrow's page is different from today's because the coach reads different data. No template, no fixed slots.
  • Adaptations + proposals + mode overrides become topic kinds too (`plan_proposal`, `mode_override`, `plan_bumped`). Same model, more cards.

What I want to know

  • Card aesthetic OK? They should feel like watch-face "complications" — one number dominates, color carries meaning, sparse copy.
  • Order of cards? Right now: order coach raised them. Alternative: order by priority (action-required first).
  • Reply chips style — Bebas all-caps work?
  • Run recap at the bottom OK? Or do you want it above the cards?
