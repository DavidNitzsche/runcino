Re-audit findings · wire-schema mismatches caught

The audit caught 4 field-name mismatches between my POST bodies and the actual route handlers. All would have 400'd on first hit. Fixed in commit 6d09c46b:

EndpointWas (wrong)Now (matches route)
POST /api/sick{symptoms, fever}{symptoms, has_fever, started}
POST /api/strength{date, type, duration}{date, session_type, duration_min}
POST /api/cross-training{duration, hr}{duration_min, avg_hr}
POST /api/goals{type}{goal_type}

Each mapping is now documented inline in API+Toolkit.swift so a future drift gets caught at code review.

Remaining toolkit gaps (not blockers · queued)

Built as code but not yet wired into a specific iOS surface. Each is ready to drop in when the surface design lands · the toolkit atom + the state-binding pattern are already there.

  • ProvenanceLine · ready to slot under any physiology number (LTHR / HRmax / VDOT / weight) on HealthView and ProfileView. The atom honors the design spec; HealthView's current LTHR/HRmax tile renders the value alone.
  • WhatChangedExpander · TrainView could swap its plan-freshness ticker for the expander · expander reads plan_adapt_* rows from /api/coach/intents and counts them.
  • HRTargetPill · Day Detail (the WeekAheadView push) could surface this when prescription.hrTargets is set. Today renders pace only.
  • ConditionsLine · one component for Today / Run Detail / Race Detail. Today already has the HOTTER tag; the toolkit consolidation would replace it with the canonical chip.
  • RaceResultHero · CountdownLadder · GelMileMarkers · CourseAnnotations · VDOTPredictionTable · RaceLogisticsTile · ProjectionSparkline · all 7 race atoms are built. RaceDayView can adopt incrementally on the next race-surface design pass.
  • StateChangeToast · race retro recalc surfacing. Fires off coach_intents (vdot_auto_recalc · lthr_auto_calibrated); requires a host surface (race retro flow not yet built on iPhone).
  • ArticleIndexCard · Learn index browser. iPhone already ships the reader; the index requires a new /learn tab/route which is a separate scope.
  • NewGoalSheet · built but no entry point yet on iPhone (Targets tab is race-only today). Add a "+ New goal" button on Targets to expose.
  • ProfileGapCard · built. Wires once briefing emits profile_gap topics on /api/briefing (currently the topics array doesn't include it for iPhone).
  • PostRunCheckinChips · built. Wires when Today shows the post-run hero (current Today only shows the pre-run hero); add to the completed-run surface.
