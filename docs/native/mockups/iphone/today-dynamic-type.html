Hierarchy survival at 1.0× (baseline):
  • Eye lands on "Tuesday" first (34pt) · then "Cruise Intervals" (22pt) · then "6 × 800m @ 6:31" (22pt orange) · then coach line · then recovery
  • Three distinct visual tiers · clear primary/secondary/tertiary
  • Fits on screen with no scroll required
Hierarchy survival at 1.5× (the common accessibility default):
  • "Tuesday" 51pt, "Cruise Intervals" 33pt — still distinct · ✓
  • Coach line reflows to 2 lines · still readable, no truncation · ✓
  • Page now requires a small scroll to reach Start button · ✓ acceptable on iPhone (vertical scroll is native)
  • Orange structure block stays dominant · ✓
Hierarchy survival at 2.0× — fixes needed in SwiftUI:
  • "Tuesday" 68pt — borders on overwhelming · acceptable per HIG (large-title scales to bounds) but consider clamping at 1.8× via .dynamicTypeSize(.large ... .xxLarge) for the large-title only
  • "Cruise Intervals" + "Threshold endurance · controlled" stack vertically with breathing room · ✓ no collapse
  • Coach line reflows to 3 lines · still readable but consumes ~25% of viewport · acceptable since it's the actionable insight
  • Recovery row stays in a single row but labels SLEEP / RHR / HRV start to crowd · fix: switch to vertical stack of three rows at >1.8× Dynamic Type
  • Start button text shrinks to fit available height — at 2× it still reads · ✓
  • The entire view now requires 1.5 page-heights of scroll · ✓ user expectation at this scale
SwiftUI implementation notes for Dynamic Type compliance:
  • Use Font.title.weight(.bold)-style declarations · they auto-scale with Dynamic Type
  • For the structure block ("6 × 800m @ 6:31"), use .dynamicTypeSize(.large ... .accessibility2) to cap maximum scaling so the orange block doesn't dominate the viewport at 3×
  • Recovery row uses ViewThatFits with two layouts: horizontal at small/medium DT, vertical stack at XL+
  • Coach line never truncates · should always reflow to additional lines
  • Start button uses .minimumScaleFactor(0.8) as a safety so "START WORKOUT" doesn't truncate at 3× even if the button height is constrained
What this stress test confirms · approved for SwiftUI implementation:
  • Hierarchy survives all three scales
  • Critical text reflows, never truncates
  • One known fix needed: recovery row → vertical stack at >1.8× DT
  • Optional: cap large-title at 1.8× via dynamic type modifier
