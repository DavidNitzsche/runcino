Watch · Always-on display rendering

Active state + AOD state side-by-side for every workout phase · designed for the dimmed state, not just dimmed from the active
← back to index
Where most watch apps fail. Apple HIG calls this out explicitly: the AOD rendering isn't a dimmed version of the active rendering — it's a separate design that respects the constraints (1Hz refresh, low brightness, OLED burn-in budget). Low-contrast colors fail, thin strokes disappear, saturated fills wash out.

Design rules applied to every phase below · enforced consistently so the runner experiences one design system in AOD:
  • Background → pure black always (saves OLED battery)
  • Hero numeral → desaturated to pure white (highest contrast)
  • State color → collapses from saturated fill to a 2px accent strip at top
  • Phase chip → preserved (which rep / what phase)
  • Footer stats → numbers kept, labels keep but at 40% white (muscle memory carries the rest)
  • Delta chip → loses background fill, becomes tinted text only
  • Progress line → preserved (overall state at a glance)
  • Font weight → bumped to 800 across the screen (thin strokes vanish at AOD brightness)

Six AOD design principles applied here · enforced in SwiftUI
  1. Background is always pure black on AOD. System light-mode setting doesn't override this · OLED battery + outdoor glare contrast both win at black.
  2. Hero numeral desaturates to pure white. The hero is the only thing that must read at AOD brightness · white is the highest-contrast value.
  3. Color collapses to a 2px accent strip. The phase-color signal still communicates state (orange=work, green=easy/recovery, blue=warmup, purple=cooldown, amber/red=drift) but as a thin line, not a saturated fill that would burn OLED.
  4. Delta chip loses background fill on AOD. Becomes tinted text only. Saved OLED area + still carries the directional read.
  5. Footer labels shrink to 8pt at 40% opacity but are kept. Don't strip them — late-night runs, sweat, sleeve coverage are all conditions where AOD-only and muscle-memory both have to work. The labels are the safety net.
  6. Refresh rate · HR / pace / cadence at 1Hz (watchOS AOD constraint). Elapsed time at 1Hz (per-second). Distance at 0.5Hz (every 2s) is fine.
Implementation requirement: SwiftUI uses the EnvironmentValues.isLuminanceReduced property to detect AOD. Each phase view renders a conditional branch on this value · don't compute the AOD layout from the active layout at runtime · ship two distinct render paths so each can be optimized independently.
