Design notes:
  • iPhone uses skeleton screens · gray blocks where text + cards will appear · perceived performance > spinners
  • Watch uses a small spinner · less real estate, skeleton would be more noise than signal
  • Loading copy is honest · "Loading today's workout…" · not vague "Please wait"
  • If loading takes >3s, show a "still loading" hint with cancel button (not mocked but real-world timeout)
  • Watch app caches last-known workout in App Group UserDefaults · if loading fails, falls back to cached data with a stale-indicator
