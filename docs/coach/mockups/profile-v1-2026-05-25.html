What's on the page

  • Identity (top) · Avatar (initials, brand gradient), name, sex/age/city. Personal anchor.
  • Coach voice — SHOES ONLY · Per your brief, this is the one thing the coach speaks to on Profile. 2 short paragraphs naming the shoes to watch.
  • RACE FIT card (amber) · CIM 195 days out, Zoom Fly 6 risky. Three rows: current mi, projected at race, retirement cap. Coach note explains.
  • YOUR ROTATION card · 5 shoes (top 5; rest summarized in coach note). Each row: name + status pill + role + mi-bar with %used. Color-coded: fresh green, seasoned blue, watch amber, retire red.
  • PERSONAL settings · Name/sex/age/city — data entry, no coach. Height row shows the +ADD affordance (gap).
  • PHYSIOLOGY (derived) · Max HR, RHR, VO2, VDOT, Weight — each with a "derived" tag showing where the value came from (OBSERVED / 60-DAY MEAN / APPLE / RACES). The runner never enters values the system can observe.
  • CONNECTIONS · Strava, Apple Health, watch pair status. Quick verify they're all green.
  • PREFERENCES · Long-run day, quality days, units. Data entry.

What the coach did NOT touch

  • Personal data entry · just settings rows
  • Physiology values · derived from data, displayed transparently with source tags
  • Connection status · status indicators only
  • Preferences · pure data entry
  • This matches your brief: coach only where it adds value (shoes), elsewhere the page is "the guts"

State source

  • Shoes: shoes table — 7 pairs for David, with run_types + mileage + mileage_cap
  • Race for fit-context: races table (CIM next A-race)
  • Physiology: derived from health_samples (max_hr peak, resting_hr 60d mean, vo2_max, body_mass)
  • Coach: Claude sonnet-4-5, 18.0s, ~$0.01
