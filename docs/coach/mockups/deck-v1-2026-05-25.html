0 · DOCTRINE

The coach changes because the inputs change.

These mockups aren't decoration. Each state is the output of a rule the system can actually evaluate — a date math, a signal threshold, a phase boundary. If the inputs are there, the engine can produce the view. If we ever ship a mockup whose trigger we can't compute, that's a flag, not a feature.

State → trigger → what changes

Every coach state below is keyed off facts already in the database (plan_workouts, training_plans, races, strava_activities, health_samples, profile). The engine reads these on every briefing and routes to the matching voice mode + card set.

State Trigger (computable today) What the coach does
TODAY · post-run morning latest_activity.date == today AND now > activity.end + 30min Recap the run, name the week's volume hit, set up tomorrow. Reply chip: SOLID / TIRED / WRECKED.
TODAY · pre-run morning today has plan_workout AND no strava_activity today AND hour < 14 Frame what today's run is for, ground in yesterday, surface any signals worth knowing before lacing up.
TODAY · rest day plan_workouts[today].type == 'rest' Permission, not absence. Sleep / mobility / recovery framing. No "you should be running" energy.
TODAY · race day plan_workouts[today].type == 'race' OR races where date == today Tone shifts to calm + ready. No volume talk. Splits target, fueling reminder, weather, "go get it."
TRAINING · phase modes plan_phases.label for current week (BASE / BUILD / PEAK / TAPER / RACE) Phase strip ringed at current; arc shows volume curve to race; voice frames what THIS phase is about.
RACE DETAIL · proximity days_to_race bucket: >60 / 30-60 / 7-30 / 0-7 / past Far: trajectory + goal. Mid: sharpening + projection. Race week: fueling, weather, pacing, kit, alarm. Post: recap + lessons.
HEALTH · watch escalation RHR 7d > baseline + N bpm AND sleep 7d < target - 1h AND days_to_race < 21 Watch list flips amber → red. Voice elevates from "noticing" to "we should adjust." Recovery card surfaces.
RACES · A-race amplified races.priority == 'A' AND days_to_race < 120 Big A-card with goal, projection vs goal, why this race matters. Coach voice frames the season around it.
PROFILE · gap surfacing profile.height_cm IS NULL — and downstream features need it (cadence target) Coach explains what the missing field unlocks. Doesn't badger. Doesn't ask for things the system can derive (HRmax, RHR).
PROFILE · shoe alert (rare) shoe.mileage + projection_to_race > shoe.retirement_cap AND shoe is the actual race shoe ONLY then does a shoe card surface. Default profile is identity + settings — shoes live in their own section, quiet.

CRUISE INTERVALS · WHAT + WHY
"Cruise" intervals are mile or longer repeats at lactate threshold — the pace you could hold for about an hour all-out. Short floats between reps keep you near the lactate ceiling without spilling over. Daniels calls it the most time-efficient way to raise your sustainable race pace.

LEARN · WHY THESE METRICS
ⓘ
HRV · WHAT + WHY
The time variation between heartbeats, measured overnight. Higher HRV means your nervous system is recovered and ready for hard training. It's one of the best early-warning signals we have for overtraining — dips here predict bad workouts before the legs do.
Read the research →
ⓘ
RHR · WHAT + WHY
Resting heart rate trends downward as aerobic fitness improves — and elevates 3-5 bpm during volume jumps, illness brewing, or sleep deficit. A sustained 5+ bpm bump that doesn't resolve in a few days is the flag to take seriously.
Read the research →
ⓘ
VO2 MAX · WHAT + WHY
The peak oxygen your body can use per minute. The single best lab predictor of endurance ceiling. Apple's estimate isn't lab-grade but it's directionally honest — month-over-month moves are real. Yours is 61.3, top-1% for your age.
