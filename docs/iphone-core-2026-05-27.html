Top panel carries the context — race countdown, training phase, post-run debrief, recovery state — and slides over a bottom panel that's always "what today actually is." Coach voice removed — language pulled from plan, schedule, and vitals. Big numbers, dark canvas. Tactical's week strip lives at the very top, always visible. The screen is a card stack: week → context → today → tabs. Open the app and the answer is already there.

six context panels
Race · A-race in sight
Orange-red gradient. Race name + days-to-go. Default for build / peak / taper / race-week.
Quality · today's the work
Amber-red gradient. Today's intervals or tempo block. Loud — this is the session that matters.
Long · the weekend block
Cyan-blue gradient. Calm. The long run rides alone. Distance and effort, no fuss.
Post-run · nailed or missed
Green-cyan gradient. Yesterday/today's completed run as the headline. Auto-replaces race panel for ~6 hours.
Rest · the body talks
Deep-blue gradient. Vitals as the panel — sleep, HRV, RHR. The day off has a job.
Phase · off-race horizon
Purple-blue gradient. No A-race. Phase + cycle mileage + streak. The panel for fitness/habit users.

promotion rule

The top panel runs an auto-promotion rule keyed off the workout type:

quality · tempo · intervals→ workout takes the panel
long run · ≥ 12mi→ long takes the panel
rest · cross-train→ vitals take the panel
just finished run · 6hr window→ post-run takes the panel
race d ≤ 3→ race locks the panel until race-end
default→ race countdown (if any) · else phase

All deterministic. No model picks the panel — the plan picks it.

why rest is its own panel

Most apps make rest days feel like a void. Faff makes them a feature. The vitals panel gives the rest day a job — show the body's response to yesterday's work — and the next-day chip prevents "wait, what's tomorrow?" anxiety.

The single verb REST. is enough. No motivation, no inspiration, no "embrace the recovery." Just the call.

design discipline · long is different

Quality days look like work because they are work — interval bars, rep counts, target paces stacked. Long days deserve the opposite. One number. One range. The pre-long screen should feel like the moment before you tie your shoes, not a wall of data.

The cyan-blue gradient says calm. The 170px 14 says that's the thing. Mile checkpoints replace the rep block — they're not segments to hit, they're moments to fuel.

why race locks the panel

From T−72h to gun, the race overrides everything. No "today's a tempo" panel can pop up over the top — the user is already operating in race-mode emotionally, the app should match. Even a missed run promoted itself only via a footer chip during race week.

The bottom panel transforms from "today's workout" to "today's logistics" — the same slot, repurposed. Bib, weather, checklist, spectator panel. Same pattern, different content.

The promotion priority:
  1. Race day · T ≤ 0 — full takeover
  2. Race week · T ≤ 3d — race locks panel
  3. Post-run · ≤ 6h — done panel
  4. Quality / Long session today — workout panel
  5. Rest day — vitals panel
  6. Build week with race — race panel
  7. No race — phase panel

what we build monday

this is the spine.

Today tab v2. Implement the card stack: week strip (existing component, just promoted to a fixed top position) → context panel (new — six variants with the gradient set + a deterministic promotion resolver) → bottom workout panel (verb-noun head + 3-stat strip + chip row). Tab bar untouched.

The promotion resolver is the load-bearing piece. It takes the day's state — today's planned workout, hours since last completed run, race horizon, vitals — and returns one of six panel variants. Pure function. Testable. Zero LLM dependency. Same inputs → same panel every time.

The verb-noun lookup is ~30 rows. Keyed off workout.type × workout.intent for pre-run and result.plan_hit × result.hr_in_zone × result.pace_delta for post-run. Hand-curated. Easy to tune. Brand voice without the brand-voice infrastructure.

Coach voice comes back later — when it does, it doesn't fight the home screen. It lives in its own tab (Confidant pattern from v1) where the chat is the surface. Home stays glanceable. That separation is the real lesson from this pass.
