The execution
surface, specced.

The dark companion app that runs today's structured workout on the wrist. This is the build target: every state, every metric, with its watchOS source and v4 token. The rest of the product is the light v4 system; the watch is dark for OLED battery, sunlight contrast, and always-on legibility. Companion app: the paired iPhone holds auth and pushes the workout. Spec authority: docs/native/01-watchos-scoping.md.

Visual canon: docs/design/watch-app.html. That file is the current, complete set of faces (including race day) and is what to match for layout, sizing, centering, and spacing. This doc is the build spec: per-metric source + token, payload shapes, haptics, the state machine, and build status. Where the mockups below differ from watch-app, watch-app wins.

C

State machine

Forward-only cursor over the phases array. No backward jumps. Each transition fires a haptic.
Idle→ Warmup→ Work⇄Recovery→ Cooldown→ Summary→Idle Work ⇄ Recovery repeats N times (here, 6). "End interval" can skip a rep forward; the cursor never moves backward.
Pre-race→ Phase⇄Fuel cue→ Finish Race day is the same cursor, walking the race's flat pace + fuel segment list. A pace segment is a course phase with its own terrain-aware target; a fuel segment fires a gel cue + haptic, then continues. Forward-only, like intervals.
D

Haptics

The watch is glance-and-feel. Haptics are how you know without looking. Fire regardless of screen on/off.
MomentPatternWhy
3s before a work rep endsTriple tap"Get ready to ease off" so you don't overrun the rep.
Work → RecoveryDistinct transition tapThe rep is done, start the jog.
Recovery → WorkDifferent transition tapGo again. Different from the work→rec tap so they're never confused.
Pace driftSingle subtle tapCurrent pace >10s off target for >5s. Silent inside the band, no nagging.
Workout completeLong hapticCooldown done. Look at the summary.
Gel due (race)Distinct double tap + cueFuel segment reached, anchored to a phase boundary. Full-screen "Gel N + water."
Phase change (race)Transition tapNew course phase, new target pace. Brief card with the phase + target.
E

Data in, data out

iPhone is the bridge: holds auth, fetches the workout, pushes it over WatchConnectivity. The watch sends the finished workout back via HealthKit.
// GET /api/watch/today  →  pushed to watch over WCSession
{
  "name": "6×800 @ T-pace",
  "summary": "6×800 @ 6:31 · 60s rec",
  "phases": [
    { "type": "warmup",   "durationSec": 600, "targetPaceSPerMi": null },
    { "type": "work",     "durationSec": 192, "targetPaceSPerMi": 391, "tolerancePaceSPerMi": 10 },
    { "type": "recovery", "durationSec":  60, "targetPaceSPerMi": null }
    // … 5 more work + recovery pairs, then cooldown
  ]
}
// GET /api/watch/race  →  same flat-segment shape, walked by the same engine
