Capabilities · per item
Note
Any context — phone model, time, brand, instructions. Markdown supported.
Due time
When this needs to be done. Faff sends a reminder 30min before. Race-relative ("T−4h", "race morning") OR absolute time.
Priority
High (race-blocking), Medium (preferable), Low (nice). Sort + filter by priority.
Sub-items
Break complex items into steps. "Drop bag packed" → 4 sub-items (towel, post-race shoes, recovery drink, phone).
Auto-check
Faff can auto-tick items when it detects the underlying event. "Watch charged" ticks when Apple Watch reports 100% on Sat night. "Bib picked up" ticks when GPS detects you at the expo.
smart suggestions
Generated from race.type + race.distance + race.location + your.history. Marathons get gels, drop bags, pacing bands. Travel races get hotel/flight items. 5-15 suggestions per race, refreshed as race-day approaches.
templates · bulk
Add a whole checklist at once. Sources: your own from past races · Faff defaults curated for each race type · travel-race add-ons that layer on top. Templates are editable — adjust before saving as a new one.
manual + auto-check
Always-available manual input at top of modal. Items can auto-check when Faff detects the underlying event — watch charged, bib picked up (GPS at expo), gels purchased (Strava integration with shopping). Less friction, more wins.
implementation · what this enables

spectator is the moat.
checklist is the trust.

Spectator mode is the feature Faff wins on. Strava can't ship it without breaking their feed model — they're committed to public posts. No one else has the watch-takeover integration. The web tracker is the wedge (Mum doesn't need to download anything), the watch bump is the moat (only works if Faff owns the watch face). Once the spectator graph exists, the network effect compounds — every runner brings their circle, and crews start watching each other.

The checklist modal answers the "where do items come from" question with three sources: smart suggestions from race data, bulk templates from past races, manual custom items. The differentiator: auto-check. When the watch is plugged in Sat 9pm, "watch charged" ticks itself. When GPS detects you at the expo, "bib picked up" ticks. The checklist gets shorter as race day approaches without you doing the work.

Build order: Spectator (circle mgmt → live tracker → bump composer → watch takeover) is sprint 04. Checklist modal is part of race detail page in sprint 03. Auto-check integrations come incrementally with Strava + HealthKit + GPS hooks. The substrate (poster + sibling + tiles) is already there — these surfaces just compose existing components.

Deck: docs/spectator-and-checklist-2026-05-27.html · companion to system-propagation deck. Spectator panel is the flagship feature. Checklist modal answers a direct interaction question. Both lock into the existing component library — same tiles, same gradient tokens, same Bebas + Inter (with tabular-nums for data) two-font stack.
