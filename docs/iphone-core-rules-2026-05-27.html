the rules.

Round two left questions: does it slide? is the bottom panel always there? what's the headline? Round three answers them. Three rules. One pattern. Two gestures. The promotion resolver picks a default layout per day; you can drag the bottom panel up to dive into data, or pull it down to let the gradient hero breathe. Tap the hero band for a full-screen takeover. That's it.
the three rules

three things,
locked.

everything else is consequence
01
Headline rule

Today is the headline.

Always. The hero panel is whatever today is — RUN EASY, INTERVALS, GO LONG, REST, NAILED IT. The race countdown lives as a chip in the bottom panel 50 weeks of the year.

The race only takes the hero when T ≤ 3 days. That's race week. The other 350 days, today comes first.

✓ EASY · INTERVALS · LONG · REST · NAILED IT
✗ LONDON MARATHON as the daily hero
02
Slide rule

Bottom panel slides.

The bottom data panel is a draggable sheet. Three snap points: collapsed (hero breathes, data peeks ~20%), default (resolver's pick, the daily home), expanded (data fills, hero shrinks to a band ~15%).

The resolver picks the default. You drag to change focus — pull up to dive into rep splits & HR zones, pull down to let the gradient breathe. Snaps to nearest position on release.

✓ Drag up → data · Drag down → gradient · Tap hero → takeover
✗ Free-form scroll · pull-to-refresh · gesture for nav
03
Gradient rule

Gradient = state.

Each day-state has one gradient. Easy = green-fade. Quality = amber-red. Long = cyan-blue. Done = green-cyan. Rest = blue-purple. Race = orange-red. Phase = purple-blue.

You should be able to identify the day from the color before the type even renders. Tomorrow's gradient lives in tomorrow's chip — the color story is the brand.

✓ 7 gradients · 7 day-states · 1:1 mapping
✗ Same gradient for everything (race-orange always)

03 · Expanded · pull up
Hero shrinks to a band. Data takes the screen — route, HR zones, vitals grid. Drill-down without leaving Today.

The slide is the magic.

The bottom panel always shows a drag handle at the top — the 42×4px grab affordance. Drag in either direction to move between the three snap positions. Releases interpolate to the nearest snap. Springy iOS-feel physics. Tap the handle to cycle for users who don't intuit the drag.

The hero gradient peeks behind the bottom panel at every position. Color is always there. Only the proportions change.

Tap hero → full takeover.

The hero band stays tappable in all three states. Tap and it animates to 100% screen, bottom slides off, tabs disappear. The takeover is the moment for: race-week poster, long-run countdown, post-run debrief — screenshot-worthy, lockscreen-worthy.

Dismiss: tap × top-right · swipe down · auto-dismiss after 8s stillness

the exception

From T−3d to gun, the race takes the hero. Today's shakeout collapses into a one-line chip in the bottom panel. Logistics — bib, weather, checklist, spectator count — get the bottom strip. The race-day operations surface from v1 explorations lives here, just at default-collapsed.

After gun: race week mode transitions to in-race mode — full takeover, live tracker for spectators, bumps coming in. Outside the scope of this dial-in but the same pattern carries.

what we build · for real this time

three rules. three states.
seven gradients.

The rules: today is the headline (except race week) · the bottom panel slides through three snap points · gradient maps 1:1 to day-state. The promotion resolver picks the gradient + the verb + the default snap. Drag changes the snap. Tap hero expands to full-screen.

The implementation order: (1) the seven gradients as a token set in Theme.swift. (2) the verb lookup as a 30-row dictionary keyed on workout type + result data. (3) the bottom sheet using .presentationDetents([.fraction(0.15), .fraction(0.32), .fraction(0.85)]) — three snap points, SwiftUI native. (4) the resolver as a pure function. (5) tap-to-takeover as a sheet over the whole screen.

What this isn't: a homepage layout. It's a system. Once the resolver + gradients + snap points are in, every future Today-tab content type rides the same rails. Treadmill mode? Same hero pattern, different verb. Live-run? Hero stays, bottom panel becomes the run interface. Spectator mode coming in to track a race? Hero stays, bottom panel becomes the live tracker. The pattern compounds.
