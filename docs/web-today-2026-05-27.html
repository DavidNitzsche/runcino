findings · what the experiment showed

same skeleton.
different bones.

The lesson from the first try: stretching the iPhone hero gradient into a full-width banner across 1280px killed everything special about it — the gradient stopped being an intentional design object and became a marketing strip. Bounding it as a poster card + dark sibling card restored the "designed" feel while still using web's horizontal real estate.

The MISSED state translates even better on web than on mobile. The two-button choice can sit inside the gradient poster (instead of stacked below the hero), and the body grid has room to show both the recommended outcome AND the alternative simultaneously — with a real ACWR load-impact visualization in place of the duplicate context rows.

What we'd build: Next.js TODAY page in web-v2/app/today/page.tsx using the same resolver call that iPhone uses. Shared component library: Poster, SiblingContext, WeekStrip, WorkoutBreakdown, HRPlanBar, HRVSparkline, PhaseProgress, VitalsGrid. iPhone and Web stay visually identical because they share design tokens (already in globals.css) and reuse the verb table + resolver outputs.

Deck: docs/web-today-2026-05-27.html · experiment translating iPhone v2.2 patterns to web. Nod and the next move is implementing this as the actual /today route in the Next.js app.
