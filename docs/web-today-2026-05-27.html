findings · what the experiment showed

same skeleton.
different bones.

The web translation works because the iPhone pattern was structured (hero band + week strip + structured rows + cards), not gestural. Drop the slide-up — everything else holds. Hero verb stays hero-scale. Stat strip moves into the gradient band as a glass-card stat trio. Structured rows fit naturally in a wide right column. Cards stack the same way.

The MISSED state translates even better on web than on mobile. The two-button choice can sit side-by-side in the hero (instead of stacked below it), and the right column has room to show both the recommended outcome AND the alternative simultaneously — instead of collapsing the non-pref to a chip like we did on mobile.

What we'd build: Next.js TODAY page in web-v2/app/today/page.tsx using the same resolver call that iPhone uses. Shared component library: HeroBand, WeekStrip, WorkoutBreakdown, HRPlanBar, ContextRows, VitalsGrid. ~6 components total. iPhone and Web stay visually identical because they share design tokens (already in globals.css) and reuse the verb table + resolver outputs.

Deck: docs/web-today-2026-05-27.html · experiment translating iPhone v2.2 patterns to web. Nod and the next move is implementing this as the actual /today route in the Next.js app.
