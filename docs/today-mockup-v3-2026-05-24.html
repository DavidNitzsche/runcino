The model

  • The coach is the eye of the app. LLM call returns { voice, topics[] }. Voice = the coach's prose. Topics = the cards. Order = the order coach raised them. Page re-shapes per day because the coach reads different data.
  • Card library is typed + finite. 8 kinds today: cadence_experiment · sleep_deficit · next_workout · profile_gap · fun_fact · weight_trend · race_horizon · recovery_amber. Renderer maps each kind to a React component.
  • Three card visual tiers. Action cards (cadence/sleep) — large, big numbers, CTAs. Info cards (next_workout) — medium, focused content. Educational cards (fun_fact) — light, purple-tinted, FYI not actionable. Gap rows (profile_gap) — slim red, with inline +Add CTA.
  • Fun facts unlock real coach voice. Before: coach was banned from saying "HRV" because it's jargon. Now: coach uses the term naturally, fun_fact card explains it. You learn the terminology, the voice stays tight. Once you've seen a fun_fact, future runs suppress repeats (tracked in profile).
  • Profile gaps as required emissions. Missing data that limits coaching MUST emit a profile_gap topic. Persistent red affordance until fixed. Tap +Add to inline-input the value.
  • "Lock in for tomorrow" closes a coaching loop. Tap commits intent in DB → coach knows on next briefing → watch can guide live (vibrate at 168 spm) → post-run, coach checks if it landed. Real two-way coaching.

What this looks like across states

  • POST-RUN (this mockup) — top: recap, then coach voice, then cards.
  • PRE-RUN — top: coach (framing for today), then workout target card (target pace, fueling, route), then cards.
  • REST DAY — top: coach (week shape, recovery focus), then cards (next workout + fun facts + profile gaps).
  • SKIPPED — top: coach (honest acknowledgment + tomorrow framing), then cards.
  • RACE DAY / RACE WEEK — own page entirely, coach voice in race-mode register, race-specific cards (fueling plan, splits target, weather, course brief).
