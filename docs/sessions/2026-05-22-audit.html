Cross-Surface Audit — web · iPhone · watch

2026-05-22 · Directive: "do a full audit of web, iPhone, watch. Make sure they're talking on EVERYTHING, that things sync that should, colors / marking done, stats, runs — and that every health metric is added correctly and used properly. No more readiness surprises."

The one-line story

The plumbing is sound — runs sync and de-dupe across all three paths, and every Apple Health metric the phone reads makes it into the database. The real risk was one root cause wearing several hats: "today" was computed three different ways and none used the device's timezone, which is exactly how a readiness score reads the wrong day. That's now fixed on the readiness/today surfaces, plus an honest "No data" state so the watch/phone can never invent a green verdict. A handful of lower-risk items are documented below for your return rather than changed unattended.
