One re-audit finding worth flagging

The HRAlerter only fires when the iPhone HK importer is allowed background HR observation

  • enableBackgroundDelivery(for: heartRate, frequency: .immediate) requires the user to have granted HR read in HealthKit AND keep the app installed. If iOS suspends the background delivery (battery saver, app force-quit), alerts stop firing silently.
  • Mitigation: the SettingsSheet toggle should re-arm on every app foreground. Already wired via FaffApp.task.
  • Watch app's own HR ceiling alert (separate code path) continues regardless — this iPhone alert is the backup, not the primary.
