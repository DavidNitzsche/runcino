02 · color = state.

03 · verb hero.

headline shrinks, verb dominates, prescription sits underneath
The iPhone hero pattern: massive verb (RUN EASY.) with the prescription tucked underneath (Hold under 148.). On web, with the wider canvas, this becomes the coach card's top half — the verb takes 60-70px, the prescription is a 20-22px sub-headline. The voice paragraph still follows below, but the runner can read the day in 1 second from the verb alone.

04 · body row.

05 · missed state.

the day the runner skipped a workout — the honest moment
The audit's most retention-relevant move: when yesterday's quality didn't happen, the home page doesn't lie about it. Amber-yellow gradient (not red, not punishment), two-button hero: CATCH UP or MOVE ON. Plan-aware default — if load is already high, MOVE ON is recommended. If quality streak is weak, CATCH UP. One tap commits. No guilt, no fake congratulations.

06 · what to ship.

each move costed, ranked by impact-per-effort
Five moves. Each can ship independently. Color = state is the highest-leverage move; verb hero changes the identity of the page most; body row is the easiest visual cleanup; MISSED state is the biggest UX unlock but needs detection logic. Pick a combo.
moveeffortimpactnotes
color = state low
~30 min		 HIGH CSS gradient on the coach card driven by the resolved state. Six gradients in tokens, one prop on the card. The single biggest visual win — instant recognition of what kind of day it is before reading anything. Risk: zero. Reversible by removing the prop.
verb hero med
~90 min		 HIGH Promote the headline to 60-72px verb on the gradient panel, lift the first sentence of the voice into a 20-22px sub-headline ("Hold under 148." / "141 bpm dead in Z2."). Voice paragraph follows. Changes the identity of the page from "stat sheet with prose" to "verb-first dashboard."
body row low
~45 min		 MED Collapse four stat cards into one horizontal strip. Same data, denser, easier to scan as a body status read. Lower visual chaos. Worth doing alongside the verb hero — same minimalism move.
MISSED state high
~4 hours		 VERY HIGH New resolver clause + amber gradient + two-button hero + plan-aware default logic. Most retention-relevant per the audit ("turns 'shit, I broke the plan' into 'okay, here's what we do'"). But: requires missed-detection (plan.scheduled vs actual.ran) AND adapt-plan logic to actually commit the choice. Ship after the visual three.
prescription line
(folded into verb hero)		 low
part of verb hero		 MED Already exists in the voice's first sentence — just need to lift it out as a separate field. Ships with verb hero, not separately.

recommended bundle.

Ship color = state + verb hero + body row together. Three moves, ~2.5 hours of work, transforms the page identity without changing the architecture we just locked. The deterministic voice we built today feeds the verb hero (lead) and the prescription (first sentence) for free.

MISSED state lives in a separate batch. The visual three can ship tonight. MISSED needs detection + plan-adapt logic + the new two-button hero pattern, plus thought on what happens after the user taps CATCH UP (does the plan auto-reshuffle? user confirms each shifted day?). Worth doing right, not rushed.

iPhone v2.2 lifted to web — same data, same structure, sharper identity. The page now knows what kind of day it is, says it loud, and gets out of the way.
