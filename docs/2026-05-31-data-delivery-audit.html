iPhone data delivery audit · why no runs were showing

Started with David reporting "ton of issues with the iPhone app · activity logs that aren't showing up · connections that aren't working" on TestFlight build 131. Ended with the root cause identified, fixed, and swept across every Codable layer in the app. Backend was correct the whole time — 100 runs ready to serve. The phone was throwing them away during decode.

Root cause · one line

The "lenient" decoder used try c.decodeIfPresent(Int.self, ...) for avg_hr / cadence / elev_gain_ft / dow. That tolerates missing key + null value but throws on a type mismatch. Apple Watch and HealthKit averaging emit fractional HR ("avg_hr": 142.5), which is JSON-valid but trips Int.self. One throw inside a LogRun fails the parent [LogRun] decode; the outer try? at LogWeek swallows it; the whole week's runs collapse to []. Every multi-source week rendered as a header with no rows beneath. David saw "0 SO FAR" while the server returned 100.

Why the "lenient" decoder wasn't lenient

The previous round (commit d68be4bb "bulletproof LogState decoder") used the pattern:

self.avg_hr = try c.decodeIfPresent(Int.self, forKey: .avg_hr)

Three failure modes for that line:

  • Key missing · decodeIfPresent returns nil. ✓ Tolerated.
  • Value is null · decodeIfPresent returns nil. ✓ Tolerated.
  • Value is 142.5 (Double instead of Int) · decodeIfPresent THROWS .typeMismatch. ✗ Not tolerated.

Apple Watch averaging windows and HealthKit conversions (e.g. meters → feet, * 3.28084) routinely emit fractional values. The JS server builds avg_hr: Number(a.avgHr) || null — JS Number() doesn't distinguish Int from Float, so a 142.7 averaged HR sails through to the JSON as 142.7. The iPhone sees a JSON number, reads Int.self, and throws.

The fix is a sibling to the existing FlexibleDouble wrapper that handles Postgres NUMERIC-as-string drift. An extension on KeyedDecodingContainer: 

extension KeyedDecodingContainer {
    func decodeFlexInt(forKey key: Key) -> Int? {
        if let i = try? decode(Int.self, forKey: key) { return i }
        if let d = try? decode(Double.self, forKey: key), d.isFinite { return Int(d.rounded()) }
        return nil
    }
}

Mirrors the server's Number(x) || null JS coercion exactly. Every JS-number shape on the wire survives.
