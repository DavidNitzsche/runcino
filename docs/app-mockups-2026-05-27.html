the visual system.

One language across all five surfaces. The runner learns it once on /today and reads every other surface for free. Specifically:

  • Color = state. Green = done / good / strong. Amber = quality / building / warn. Cyan = long / cool data. Purple = easy. Orange = race. Red = bad / spike. Always the same meaning.
  • Display font does the lifting. Helvetica Neue Bold at 32-80px for any number worth reading. Inter at 12-15px for prose. Oswald uppercase at 9-11px for labels and eyebrows.
  • Status dots beat sentences. A red dot in the corner of a tile says everything a sentence does in 1/30th the time. Use them everywhere the user needs to scan.
  • Scoreboard headlines. EASY RUN DONE / BUILD WEEK 6 OF 8 / 81 DAYS TO LONDON / STRONG MORNING / DAVID NITZSCHE. Caps, big, declarative.
  • Top nav stays minimal. Five tabs, no overflow. Active tab gets the surface-appropriate accent color underneath (green / amber / orange / purple / cyan).
  • Two-column main where there's a "now" and an "around it". /today and /me lean two-col. /plan and /races and /health lean one-col-with-stacks because their content is sequential.

Ship order from here: /plan first (phase + week list lives close to plan-aware coach moves we already have), then /races (race-orange identity), then /health (sparklines need 14d data assembled), then /me (mostly existing components re-skinned). Each one is a 2-4 hour ship if we don't redesign mid-flight.
