The prefetch pattern (codified)

For every future fetch-on-open modal

1. Modal accepts prefetched* props. Initializes its useState / @State synchronously from them. First paint is complete when warm.

2. Parent owns the cache. Date-keyed Map for list-of-things, module-scope singleton for app-wide things (settings, articles).

3. Trigger fires prefetch on intent. onMouseEnter + onFocus on web; pre-warming via .task on iPhone.

4. Mount-time prefetch when the trigger is highly visible. Avatar always visible → fetch settings on layout mount. Current week always read → batch-fetch on log mount.

5. Fall back to on-mount fetch when cache is cold. Never break old behavior; props are optional.

// Web shape
function Modal({ prefetchedData, ... }) {
  const [data, setData] = useState(prefetchedData ?? null);
  const [loading, setLoading] = useState(prefetchedData == null);
  useEffect(() => {
    if (prefetchedData) { setData(prefetchedData); setLoading(false); return; }
    fetch(url).then(...);
  }, [prefetchedData]);
}

// iPhone shape
struct Sheet: View {
  let prefetched: T?
  init(... prefetched: T? = nil) {
    self.prefetched = prefetched
    _data = State(initialValue: prefetched)
    _loading = State(initialValue: prefetched == nil)
  }
  .task { if prefetched == nil { await load() } }
}

Apply this pattern to every new modal we build. Add it to CLAUDE.md if it isn't there yet.
